# 朝活状況 視覚化ダッシュボード — 完全構築ガイド

> Discordのボイスチャンネルに入っている人が、カフェの2D画像の上にアバターとして座って表示されるWebページを、ゼロから作って公開するまでの手順書です。

---

## 全体の流れ

```
STEP 1: 必要なツールをインストールする
STEP 2: Discord Botを作る
STEP 3: Claude Codeでプロジェクトを作る
STEP 4: 背景画像を用意して設定する
STEP 5: 座席位置を設定する
STEP 6: 見た目を調整する
STEP 7: ローカルで動作確認する
STEP 8: GitHubにコードを保存する
STEP 9: Renderで本番公開する
```

---

## STEP 1: 必要なツールをインストールする

### 1-1. Node.js をインストール

Node.jsはサーバーを動かすために必要です。

1. https://nodejs.org/ を開く
2. 「LTS」と書かれた方のボタンをクリックしてダウンロード
3. ダウンロードしたファイルを開いてインストール（全部「次へ」でOK）

確認方法（ターミナルを開いて入力）:
```bash
node -v
```
`v18.x.x` や `v20.x.x` のような数字が出ればOK。

### 1-2. Claude Code をインストール

Claude Codeはターミナルで動くAIアシスタントです。
コードを書く指示を日本語で出せます。

```bash
npm install -g @anthropic-ai/claude-code
```

インストール後、初回起動時に認証が求められます:
```bash
claude
```
画面の指示に従ってAnthropicアカウントでログインしてください。

### 1-3. Git をインストール（入っていない場合）

Macの場合:
```bash
xcode-select --install
```

Windowsの場合:
https://git-scm.com/ からダウンロードしてインストール

確認方法:
```bash
git --version
```

## STEP 2: Discord Botを作る

### 2-1. Botアプリケーションを作成する

1. https://discord.com/developers/applications を開く（Discordアカウントでログイン）
2. 右上の「New Application」をクリック
3. 名前を入力（例：「朝活ダッシュボード」）→「Create」

### 2-2. Bot Tokenを取得する

1. 左メニューの「Bot」をクリック
2. 「Reset Token」ボタンを押す → 確認画面で「Yes, do it!」
3. 表示されたトークン（長い英数字の文字列）をコピーしてメモ帳に保存

> **重要**: このトークンは二度と表示されません。必ず保存してください。
> 他の人に見せると乗っ取られるので、絶対に共有しないでください。

### 2-3. Bot の権限を設定する

同じ「Bot」ページの下の方にある「Privileged Gateway Intents」で以下の2つをオンにする:
- **SERVER MEMBERS INTENT** → スイッチをオンに
- **PRESENCE INTENT** → スイッチをオンに

「Save Changes」を押す。

### 2-4. Botをサーバーに招待する

1. 左メニューの「OAuth2」をクリック
2. 「OAuth2 URL Generator」のSCOPESで「bot」にチェック
3. BOT PERMISSIONSで以下にチェック:
   - **Connect**
   - **View Channels**
4. 一番下に表示されるURLをコピー
5. そのURLをブラウザで開く
6. Botを追加したいサーバーを選んで「認証」

### 2-5. サーバーIDとチャンネルIDを取得する

1. Discordアプリを開く
2. 左下の歯車アイコン（ユーザー設定）→「アプリの設定」→「詳細設定」
3. 「開発者モード」をオンにする
4. 設定画面を閉じる

IDの取得:
- サーバー名を**右クリック** →「サーバーIDをコピー」→ メモ帳に保存（**Guild ID**）
- 監視したいボイスチャンネルを**右クリック** →「チャンネルIDをコピー」→ メモ帳に保存（**Channel ID**）

### ここまでのメモ確認

以下の3つがメモできていればOK:
```
Bot Token:    MTQ4ODQ2......（長い文字列）
Guild ID:     1331102548571983963（数字）
Channel ID:   1344497188457418824（数字）
```

## STEP 3: Claude Codeでプロジェクトを作る

### 3-1. 作業フォルダを作成してClaude Codeを起動

ターミナルを開いて以下を入力:

```bash
cd ~/Desktop
mkdir 朝活ダッシュボード
cd 朝活ダッシュボード
claude
```

### 3-2. Claude Codeにプロジェクト作成を指示する

以下をそのままコピペして送信してください:

```
Discordのボイスチャンネルの参加者を、2D画像の上にアバターとして表示する
Webダッシュボードを作ってください。

【技術構成】
- バックエンド: Node.js + Express + discord.js
- フロントエンド: HTML/CSS/JS（フレームワークなし）
- 5秒ごとにAPIをポーリングして参加者を更新

【機能】
- Discord Botでボイスチャンネルの参加者を取得するAPIを作成
- GET /api/participants で参加者一覧をJSONで返す
- 参加者のアバターと名前を背景画像の上にランダム配置で表示
- 入退室時にアニメーション（入室: ポップイン、退室: フェードアウト）
- 席を配列で定義して、ランダムに座らせる（重複なし、退室時に席を解放）
- 名前の横に緑色のオンラインマークを表示

【環境変数】
.envファイルに以下を設定:
DISCORD_BOT_TOKEN=（あとで設定）
DISCORD_GUILD_ID=（あとで設定）
DISCORD_VOICE_CHANNEL_ID=（あとで設定）
PORT=3000

【デザイン】
- 背景画像の上にアバターを重ねるレイヤー構造
- 左上にすりガラス風の参加状況パネル（参加人数を表示）
- レスポンシブ対応（PC/スマホ両対応）

package.jsonのscriptsに start と dev コマンドを用意してください。
```

→ Claude Codeがファイルを自動生成します。完了するまで待ちます。

### 3-3. .envファイルを設定する

Claude Codeに以下を送信（STEP 2でメモした値を貼り付けてください）:

```
.envファイルを作成して以下を設定してください:

DISCORD_BOT_TOKEN=ここにBotトークンを貼り付け
DISCORD_GUILD_ID=ここにGuild IDを貼り付け
DISCORD_VOICE_CHANNEL_ID=ここにChannel IDを貼り付け
PORT=3000
```

## STEP 4: 背景画像を用意して設定する

### 4-1. 背景画像を用意する

カフェ、教室、オフィスなどの**上から見下ろした（俯瞰/アイソメトリック）イラスト**がおすすめです。

AIで画像生成する場合のプロンプト例:
```
アイソメトリックビューのおしゃれなカフェの内装、
テーブルと椅子が複数配置されたコワーキングスペース風、
温かみのある色合い、植物やパソコンが置かれている、
16:9の横長比率、イラスト調
```

### 4-2. 背景画像をプロジェクトに設定する

画像ファイルを `public/` フォルダにコピーしてから、Claude Codeに指示:

```
背景画像を public/（ファイル名）に変更してください
```

## STEP 5: 座席位置を設定する

### 5-1. サーバーを起動する

Claude Codeに:
```
サーバーを起動してください
```

または自分でターミナルに入力:
```bash
npm start
```

### 5-2. 座席位置を決める

1. ブラウザで `http://localhost:3000` を開く
2. スクリーンショットを撮る
3. ペイントツール（Macは「プレビュー」、Windowsは「ペイント」）で開く
4. 椅子がある場所に**番号付きの丸**を書き込む（1, 2, 3, 4...）

5. その画像をClaude Codeに送って、こう伝える:

```
この配置図の通りに座席を設定してください
```

→ Claude Codeが画像を読み取って、座標を自動でコードに反映してくれます。

### 5-3. ずれていた場合

ブラウザで確認して、位置がずれていたら:

```
座席の位置がずれているので確認して調整してください
```

## STEP 6: 見た目を調整する

以下の指示をClaude Codeに送るだけで調整できます。
必要なものだけ使ってください。

### アバターの遠近感（奥は小さく手前は大きく）
```
奥にいる人は小さく、手前にいる人は大きく表示されるように
遠近感をつけてください
```

### スマホ対応
```
スマホの縦持ちで見たときに、画面の高さに合わせて表示して、
横スクロールで全体が見えるようにしてください
```

### パネルのデザイン変更
```
左上の参加状況パネルのデザインを変えてください。
（具体的にどう変えたいか書く）
```

### その他の例
```
アバターのサイズをもっと大きくしてください
背景画像の透明度を下げてください
名前のフォントサイズを大きくしてください
```

## STEP 7: ローカルで動作確認する

### 7-1. サーバーを起動

```bash
npm start
```

### 7-2. ブラウザで確認

`http://localhost:3000` を開く。

### 7-3. 実際にボイスチャンネルに入ってもらう

1. Discordで監視対象のボイスチャンネルに誰かに入ってもらう
2. 5秒ほど待つ
3. ブラウザにアバターが表示されれば成功！

### うまくいかない場合

Claude Codeにそのまま伝えてください:

```
ボイスチャンネルに人が入っているのにアバターが表示されません。
原因を調査して修正してください。
```

## STEP 8: GitHubにコードを保存する

GitHubにコードを保存しておくと、バックアップになり、
次のSTEP 9のデプロイにも必要です。

### 8-1. GitHubアカウントを作る（持っていない場合）

1. https://github.com を開く
2. 「Sign up」からアカウントを作成（メールアドレスが必要）

### 8-2. 新しいリポジトリを作成する

1. GitHubにログイン
2. 右上の「+」→「New repository」をクリック
3. Repository name に名前を入力（例：`morning-room`）
4. **Private**（非公開）を選択（Botトークンの漏洩を防ぐため）
5. それ以外はそのままでOK
6. 「Create repository」をクリック
7. 表示されるURLをコピー（例：`https://github.com/あなたの名前/morning-room.git`）

### 8-3. Claude Codeでプッシュする

Claude Codeに以下を送信（URLは自分のものに置き換え）:

```
Gitを初期化して、GitHubにプッシュしてください。
リモートURLは https://github.com/あなたの名前/morning-room.git です。
.envファイルはgitignoreに入れてプッシュしないでください。
```

> **重要**: `.env` ファイル（Botトークンが入っている）は絶対にGitHubにプッシュしないでください。Claude Codeが `.gitignore` に自動で追加してくれます。

初回プッシュ時にGitHubの認証を求められることがあります。
画面の指示に従ってログインしてください。

## STEP 9: Renderで本番公開する

Renderは無料でNode.jsアプリを公開できるサービスです。
公開すると、スマホや他のPCからもURLでアクセスできるようになります。

### 9-1. Renderアカウントを作る

1. https://render.com を開く
2. 「Get Started for Free」をクリック
3. 「GitHub」でサインアップするのが一番楽です（STEP 8のアカウントと連携される）

### 9-2. 新しいWebサービスを作成する

1. ダッシュボードの「New +」→「Web Service」をクリック
2. 「Build and deploy from a Git repository」を選択 →「Next」
3. GitHubリポジトリの一覧から自分のリポジトリ（例：`morning-room`）を選択 →「Connect」

> リポジトリが表示されない場合は「Configure account」からGitHubとの連携を設定してください。

### 9-3. サービスを設定する

以下の通りに入力してください:

| 設定項目 | 入力内容 |
|---------|---------|
| Name | 好きな名前（例：`asakatsu-dashboard`） |
| Region | `Singapore (Southeast Asia)` ← 日本から一番近い |
| Branch | `main` |
| Runtime | `Node` |
| Build Command | `npm install` |
| Start Command | `npm start` |
| Instance Type | `Free`（無料プラン） |

### 9-4. 環境変数を設定する

ページ下部の「Environment Variables」セクションで「Add Environment Variable」を押して、以下の4つを**1つずつ**追加する:

| Key | Value |
|-----|-------|
| `DISCORD_BOT_TOKEN` | STEP 2でメモしたBotトークン |
| `DISCORD_GUILD_ID` | STEP 2でメモしたGuild ID |
| `DISCORD_VOICE_CHANNEL_ID` | STEP 2でメモしたChannel ID |
| `PORT` | `3000` |

### 9-5. デプロイする

「Create Web Service」ボタンを押す。

数分待つと、デプロイが完了して画面上部にURLが表示されます。
（例：`https://asakatsu-dashboard.onrender.com`）

そのURLをブラウザで開けば、世界中どこからでもアクセスできます！

### 9-6. コードを更新したいとき

Claude Codeで修正した後:

```
コミットしてプッシュしてください
```

と伝えるだけでOK。GitHubにプッシュすると、Renderが自動でデプロイしてくれます。

### 9-7. 無料プランの注意点

- **スリープ機能**: 15分間アクセスがないと自動でスリープします。
  次にアクセスしたときに起動まで30〜60秒ほどかかります。
- **月750時間**: 無料枠は月750時間まで。個人利用なら十分です。

## ファイル構成（参考）

```
プロジェクト/
├── .env                  ← 秘密情報（Bot Token等）※GitHubに上げない
├── .gitignore            ← .envなどを除外する設定
├── package.json          ← プロジェクト設定・依存パッケージ
├── server.js             ← サーバー本体（Discord Bot + Express API）
└── public/               ← ブラウザに表示されるファイル
    ├── index.html        ← Webページの構造
    ├── styles.css        ← 見た目のスタイル
    ├── app.js            ← 座席配置・アバター表示の処理
    └── bg-cafe.jpg       ← 背景画像
```

## 困ったときは

Claude Codeは日本語で何でも聞けます。
「何がわからないかわからない」ときは、こう送ってみてください:

```
今の状態を確認して、何が問題か教えてください
```
